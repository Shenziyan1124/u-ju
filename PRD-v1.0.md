# U Ju 活动管理平台 — 产品需求文档 (PRD) v1.0

## 1. 产品概述

### 1.1 产品定位

U Ju 是一个综合性活动管理平台，连接活动发布者（企业/机构）与活动参与者（个人用户），提供从活动发布、浏览、报名到支付的全流程服务。

### 1.2 产品目标

- 为发布者提供便捷的活动发布和管理工具
- 为参与者提供丰富的活动发现和报名体验
- 构建安全可靠的在线支付和退款体系

### 1.3 目标用户

| 用户类型 | 描述 | 核心需求 |
|----------|------|----------|
| 参与者 | 普通个人用户 | 发现活动、报名参加、管理票券 |
| 发布者 | 企业/机构用户 | 发布活动、管理报名、查看数据 |
| 管理员 | 平台运营人员 | 审核管理、数据统计、系统配置 |

## 2. 系统架构

### 2.1 三端架构

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  u-ju-uniapp    │    │  u-ju-admin     │    │  u-ju-server    │

└────────┬────────┘    └────────┬────────┘    └────────┬────────┘
         │                      │                      │
         └──────────────────────┴──────────────────────┘
                                │
                    ┌───────────┴───────────┐
                    │  MySQL + Redis        │
                    │  数据库 + 缓存        │
                    └───────────────────────┘
```

### 2.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 移动端 | UniApp (Vue3) | 一套代码，小程序+H5 |
| 后台 | Vue3 + Element Plus | PC端管理系统 |
| 后端 | Node.js + Express | RESTful API |
| 数据库 | MySQL 8.0 | 关系型数据存储 |
| 缓存 | Redis | 会话、验证码、热点数据 |
| 认证 | JWT | Token 认证机制 |
| 支付 | 微信支付 + 支付宝 | 多渠道支付 |
| 文件存储 | 本地/阿里云 OSS | 图片和文件上传 |

## 3. 功能模块详述

### 3.1 用户模块

#### 3.1.1 注册

**参与者注册：**
- 必填：用户名、手机号、密码、真实姓名
- 选填：邮箱、性别、出生日期、头像
- 验证：手机短信验证码
- 规则：密码至少8位，包含大小写字母和数字
- 注册成功自动登录

**发布者注册：**
- 先完成参与者注册
- 再提交企业认证：企业名称、统一社会信用代码、营业执照、企业地址、联系人
- 管理员审核通过后获得发布者权限

#### 3.1.2 登录

- 手机号 + 密码登录
- 手机号 + 短信验证码登录
- 记住登录状态

#### 3.1.3 个人资料

- 查看/编辑基本信息
- 修改密码
- 头像上传

### 3.2 活动模块

#### 3.2.1 活动发布（发布者）

**基本信息：**
- 活动标题（1-100字）
- 活动封面图（支持多图）
- 活动分类：会议论坛/培训讲座/展览展会/体育赛事/文艺演出/社交聚会/其他
- 活动标签（最多5个）
- 活动简介（10-200字）
- 活动详情（富文本）

**时间地点：**
- 活动开始/结束时间
- 报名截止时间（早于开始时间）
- 地点类型：线下/线上/混合
- 线下地址：省市区 + 详细地址

**报名设置：**
- 免费/付费
- 付费活动支持多票型（早鸟票/标准票/VIP票）
- 每种票型：名称、价格、数量、说明
- 人数限制：最少/最多参与人数
- 报名审核：自动通过/需要审核
- 报名表单：默认字段(姓名/手机/邮箱) + 自定义字段

**其他设置：**
- 活动状态：草稿/已发布/已下架/已取消
- 是否公开展示
- 联系方式
- 退款政策

#### 3.2.2 活动浏览（参与者）

**首页展示：**
- 精选推荐
- 热门活动
- 最新发布
- 即将开始

**筛选排序：**
- 分类筛选
- 时间筛选
- 地点筛选
- 价格筛选（免费/付费）
- 关键词搜索
- 排序：最新/最热/即将开始/价格

#### 3.2.3 活动详情

- 完整活动信息展示
- 主办方信息
- 已报名人数/剩余名额
- 报名按钮（动态状态）
- 收藏功能
- 分享功能

### 3.3 报名模块

#### 3.3.1 免费活动报名

1. 点击"立即报名"
2. 填写报名表单
3. 确认提交
4. 等待审核（如需）或直接成功

#### 3.3.2 付费活动报名

1. 选择票型和数量
2. 填写报名表单
3. 确认订单
4. 支付
5. 报名生效，获取电子票

#### 3.3.3 报名管理

- 查看报名记录
- 筛选状态：待审核/已通过/已拒绝/已取消
- 取消报名
- 申请退款

### 3.4 支付模块

- 支付方式：微信支付、支付宝
- 订单状态：待支付/支付中/已支付/已取消/已退款/退款中
- 15分钟超时自动取消
- 退款规则由发布者设定
- 退款原路返回

### 3.5 通知模块

- 站内消息
- 类型：系统通知/活动通知/支付通知
- 未读消息提醒

### 3.6 后台管理模块

- 用户管理：列表/编辑/状态管理/认证审核
- 活动管理：列表/审核/违规处理/推荐设置
- 订单管理：列表/详情/退款审核
- 内容管理：轮播图/公告/帮助文档
- 系统设置：基础配置/支付配置/短信配置

## 5. 非功能性需求

- 页面加载 < 3秒
- API 响应 < 500ms
- JWT Token 认证
- 密码 bcrypt 加密
- HTTPS 通信
- XSS/CSRF 防护
- 响应式设计（320px - 2560px）

## 6. MVP 优先级

### P0 — 必须实现

1. 用户注册登录
2. 活动发布（基础信息 + 免费/付费）
3. 活动浏览搜索
4. 活动报名（免费 + 付费）
5. 用户中心
6. 后台管理（基础）

1. 企业认证审核
2. 多票型支持
3. 报名审核
4. 收藏功能
5. 消息通知

### P2 — 可以后续实现

1. 评价系统
2. 签到管理
3. 优惠码
4. 数据统计图表
5. 问答功能
